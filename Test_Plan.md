# Test Plan

## Project Name
E-Commerce Website Manual Testing

## Objective
To verify that the E-Commerce Website functions according to business requirements and provides a seamless user experience.

## Scope of Testing

The following modules will be tested:

- User Registration
- User Login
- Product Search
- Product Details
- Wishlist
- Shopping Cart
- Checkout Process
- Order Confirmation

## Testing Types

- Functional Testing
- Smoke Testing
- Sanity Testing
- Regression Testing
- UI Testing
- End-to-End Testing

## Test Environment

| Component | Details |
|------------|------------|
| Operating System | Windows 11 |
| Browser | Google Chrome |
| Application Type | Web Application |
| Testing Method | Manual Testing |

## Entry Criteria

- Application URL is accessible.
- Test environment is available.
- Requirements are documented.

## Exit Criteria

- All planned test cases executed.
- Critical defects resolved.
- Test summary report prepared.

## Deliverables

- Test Cases
- Test Scenarios
- Bug Report
- RTM
- Test Summary Report
- Screenshots

## Risks

- Unavailable test environment
- Requirement changes
- Browser compatibility issues

## Assumptions

- Stable internet connection.
- Test data is available.
- Application is deployed successfully.

## Approval

Testing completed successfully based on the defined scope and test objectives.
